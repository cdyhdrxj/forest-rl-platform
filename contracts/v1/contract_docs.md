# Журнал эксперимента

Контракт описывает передачу данных об одном эпизоде эксперимента: общая статистика и последовательность событий.

Эпизод включает:
- агрегированные показатели (success, reward_total, path_cost, coverage_ratio)
- список событий, произошедших в ходе эпизода (столкновения, достижение цели, появление и обнаружение нарушителей, пожары)

## Верхний уровень

- **run_id** *(integer, required)*  
  ID запуска эксперимента

- **episode** *(object, required)*  
  Объект с общей информацией об эпизоде

- **events** *(array)*  
  Список событий, произошедших в эпизоде

## Эпизод (episode)

- **episode_index** *(integer, required)*  
  Порядковый номер эпизода в рамках запуска

- **seed** *(integer)*  
  seed генератора случайных чисел, использованное для этого эпизода

- **success** *(boolean)*  
  Признак успешного завершения эпизода

- **terminated_by** *(string)*  
  Причина завершения: `goal_reached`

- **reward_total** *(number)*  
  Суммарная награда за эпизод

- **steps_count** *(integer)*  
  Количество шагов в эпизоде

- **duration_sec** *(number)*  
  Продолжительность эпизода в секундах симуляционного времени

- **path_length** *(number)*  
  Длина пройденной траектории

- **path_cost** *(number)*  
  Стоимость траектории

- **collisions_count** *(integer)*  
  Количество столкновений

- **coverage_ratio** *(number)*  
  Доля покрытия территории (актуально для патрулирования), значение от 0 до 1

- **avg_detection_delay** *(number)*  
  Средняя задержка обнаружения событий (в секундах)

- **total_damage** *(number)*  
  Суммарный ущерб, накопленный за эпизод

## Событие (events)

- **event_type** *(string, required)*  
  Тип события. Возможные значения:  
  `collision` - столкновение с препятствием  
  `goal_reached` - достижение целевой точки  
  `intruder_spawn` - появление нарушителя  
  `intruder_detected` - обнаружение нарушителя  
  `intruder_intercepted` - нейтрализация нарушителя 
  `fire_started` - начало пожара  
  `fire_finished` - окончание пожара  
  `damage`- нанесён ущерб

- **step_index** *(integer)*  
  Номер шага, на котором произошло событие

- **sim_time_sec** *(number)*  
  Момент времени в симуляции (секунды от начала эпизода)

- **x**, **y**, **z** *(number)*  
  Координаты места события

- **payload** *(object)*  
  Дополнительные данные события в свободном формате

# Описание сценария

Контракт описывает конфигурацию лесной среды для воспроизводимых экспериментов.

Сценарий включает:
- метаданные (проект, код, название, режим)
- версию с параметрами окружения (рельеф, препятствия, события, сенсоры, награды)
- ссылки на файловые слои (высоты, проходимость, препятствия, зоны событий)

## Верхний уровень

- **project_id** *(integer, required)*  
  ID проекта, к которому относится сценарий

- **code** *(string, required)*  
  Краткий уникальный код сценария в рамках проекта

- **name** *(string, required)*  
  Полное наименование сценария

- **mode** *(string, required)*  
  Режим, для которого предназначен сценарий:  
  `trail | robot | patrol | fast | multiagent`

- **description** *(string)*  
  Текстовое описание сценария

- **version** *(object, required)*  
  Объект с параметрами конкретной версии

## Версия сценария (version)

- **version_no** *(integer, required)*  
  Порядковый номер версии

- **seed** *(integer)*  
  seed генератора случайных чисел для воспроизводимости

- **is_active** *(boolean)*  
  Признак активной версии

- **terrain_config** *(object)*  
  JSON-параметры рельефа (высоты)

- **obstacle_config** *(object)*  
  JSON-параметры препятствий (расположение, размеры, свойства)

- **event_config** *(object)*  
  JSON-параметры динамических событий (пожары, нарушения, интенсивность)

- **sensor_config** *(object)*  
  JSON-конфигурация сенсоров агента (дальность, углы обзора)

- **reward_config** *(object)*  
  JSON-параметры функции награды (веса, штрафы, бонусы)

- **world_file_uri** *(string)*  
  Путь или URI к файлу мира симулятора

- **preview_image_uri** *(string)*  
  Путь или URI к изображению предварительного просмотра

- **layers** *(array)*  
  Список файловых слоёв, дополняющих сценарий

## Слой сценария (layers)

- **layer_type** *(string, required)*  
  Тип слоя:  
  `elevation` - высоты (растр)  
  `traversability`- проходимость
  `obstacles` - препятствия 
  `events` - зоны возникновения событий

- **file_uri** *(string, required)*  
  Путь или URI к файлу слоя

- **file_format** *(string)*  
  Формат файла

- **description** *(string)*  
  Пояснение к содержанию слоя


# Метрики эксперимента (metrics.schema.json)

Контракт описывает передачу метрик экспериментов.

Метрики:
- серия (metric_series)
- точки серии (metric_points)

## Верхний уровень

- **run_id** *(integer, required)*  
  ID запуска эксперимента

- **series** *(array, required)*  
  Список метрик (каждая = одна серия)

## Серия метрики

- **name** *(string, required)*  
  Название метрики

- **unit** *(string)*  
  Единица измерения

- **aggregation** *(string)*  
  Тип агрегации:
  `mean | sum | min | max`

- **source** *(string)*  
  Источник:
  `trainer | simulator | robot_adapter`

- **mode** *(string)*  
  Режим:
  `trail | robot | patrol | multiagent`

- **description** *(string)*  
  Описание метрики

- **points** *(array, required)*  
  Временной ряд значений

## Точка метрики

- **point_index** *(integer)*  
  Порядковый номер точки

- **train_step** *(integer)*  
  Шаг обучения

- **episode_index** *(integer)*  
  Номер эпизода

- **wall_time_sec** *(number)*  
  Время с начала запуска (в секундах)

- **value** *(number, required)*  
  Значение метрики
