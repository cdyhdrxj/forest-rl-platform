# Интерфейсы ROS 2

Здесь фиксируются топики, сервисы и форматы сообщений, используемые в проекте.

## 1. Общие принципы

- Пространство имён: отсутствует (топики без префикса) (В будущем будет измененено)
- Время: ROS time
- Все ключевые события среды публикуются как сообщения

## 2. Топики

### 2.1 Наблюдения (среда -> агент)

#### `/freight/base_scan` (будет изменено)
**Тип:** `sensor_msgs/LaserScan`  
**Описание:** Данные лидара робота.  
**Издатель:** Unity симулятор  
**Подписчики:** RL агент

#### `/model_pose`
**Тип:** `geometry_msgs/PoseStamped`  
**Описание:** Положение и ориентация робота.  
**Издатель:** Unity симулятор  
**Подписчики:** RL агент

#### `/goal_pose`
**Тип:** `geometry_msgs/PoseStamped`  
**Описание:** Целевая позиция для робота.  
**Издатель:** Unity симулятор  
**Подписчик:** RL агент

### 2.2 Действия (агент -> среда)

#### `/cmd_vel`
**Тип:** `geometry_msgs/Twist`  
**Описание:** Команда движения робота (непрерывное управление).
**Издатель:** RL агент
**Подписчик:** Unity симулятор

### 2.3 Управление (агент -> среда)

#### `/reset_robot`
**Тип:** `std_msgs/Empty`  
**Описание:** Сброс робота в начальное положение.  
**Издатель:** RL агент  
**Подписчик:** Unity симулятор


## 3. Форматы данных

### 3.1 Наблюдение (observation)

Вектор из 13 значений (float32):
- 9 секторов лидара (расстояния до препятствий, нормированы 0–10)
- 2 координаты позиции робота (x, y)
- 2 координаты цели (x, y)

### 3.2 Награда (reward)

Вычисляется внутри RL агента.

## 3. Мост ROS

Для связи с Unity используется **rosbridge** (WebSocket).

**Адрес подключения:** `ws://localhost:9090`
