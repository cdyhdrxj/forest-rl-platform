# Интерфейсы ROS 2

Версия: v1

## 1. Общие принципы

1. Интерфейсы совместимы с 3 видами среды:
    - 3D среда
    - 2D непрерывная среда
    - Клеточная среда
2. Каждый агент работает в своем пространстве имён `/robot_{id}/`.
    - Для режима с одним агентом используется `robot_0`.
    - Для режима с $n$ агентами используются `robot_0`, `robot_1`, $\ldots$, `robot_{n-1}`.
3. Для остальных событий используется пространство имён `/env/`.
4. Ключевые происшествия (столкновение, достижение цели и пр.) публикуются через топик событий `/env/events`.
5. Поддержка пошагового выполнения через сервис `step` для обучения в клеточной среде.
6. Время: симуляционное время ROS


## 2. Топики: Наблюдения (Среда -> Агент)

### `/robot_{id}/base_scan`
* **Тип:** `sensor_msgs/LaserScan`
* **Описание:** Данные лидара робота.
    * **Клеточная среда:** Эмуляция лидара (TODO)

### `/robot_{id}/pose`
* **Тип:** `geometry_msgs/PoseStamped`
* **Описание:** Текущие координаты и ориентация.
    * **Клеточная среда:** Координаты текущей ячейки $(x, y)$, ориентация игнорируется.

### `/env/events`
* **Тип:** `forest_msgs/Event`
* **Описание:** Регистрация всех типов происшествий.
* **Типы событий:**
    - `COLLISION`= 0 - столкновение
    - `GOAL` = 1 - достижение цели
    - `INTRUDER` = 2 - пойман нарушитель

## 3. Топики: Управление (Агент -> Среда)

### `/robot_{id}/cmd_vel`
* **Тип:** `geometry_msgs/Twist`
* **Описание:**
    * Линейная скорость (м/с) и угловая скорость (рад/с).
    * Только **3D/2D непрерывная среда**

### `/robot_{id}/cmd_step`
* **Тип:** `forest_msgs/StepCmd`
* **Описание:**
    * Дискретное действие, переход в соседнюю по стороне клетку
    * Только **клеточная среда**
* **Виды действий**:
    * `UP` = 0 - наверх
    * `DOWN` = 1 - вниз
    * `LEFT` = 2 - влево
    * `RIGHT` = 3 - вправо
    * `STAY` = 4 - остаться в клетке

## 4. Сервисы

### `/env/reset`
* **Тип:** `std_srvs/Trigger`
* **Описание:**
    * Полная очистка сцены, возврат всех агентов в начальные точки.
    * Возвращает статус успешности операции.

### `/env/step`
* **Тип:** `std_srvs/Empty`
* **Описание:**
    * Принудительный расчет одного такта симуляции для синхронизации между агентом и средой.
    * После вызова сервиса все топики наблюдений (`/robot_{id}/pose`, `/robot_{id}/base_scan`, `/env/events`) будут обновлены до возврата управления вызывающему коду.
    * Только **клеточная среда**.

## 5. Форматы данных

### 5.1 Сообщение `forest_msgs/Event.msg`
Формат для регистрации всех типов происшествий в системе.

```text
# Заголовок со временем события
std_msgs/Header header

# ID робота
int32 robot_id

# Координаты события
geometry_msgs/Point position

uint8 COLLISION=0
uint8 GOAL=1
uint8 INTRUDER=2

# Тип
uint8 event_type

# Численное значение (для event_type=INTRUDER: ID нарушителя, иначе -1)
int32 value
```

### 5.2 Сообщение `forest_msgs/StepCmd.msg`

```text
uint8 UP=0
uint8 DOWN=1
uint8 LEFT=2
uint8 RIGHT=3
uint8 STAY=4

# Действие
uint8 action
```
