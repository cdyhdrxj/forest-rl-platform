# Документация платформы Forest RL

Этот каталог содержит актуальную markdown-документацию по текущему состоянию проекта.
Основной принцип простой: если описание в `docs/` расходится со старым `.docx`, источником правды считается текущая реализация в коде и markdown-файлы из этого каталога.

## Что здесь лежит

- `architecture/README.md` — актуальная схема компонентов и потоков данных.
- `api/README.md` — текущее устройство backend API и WebSocket-протокола.
- `scenarios/README.md` — генерация, хранение и загрузка сценариев.
- `database/README.md` — текущая модель данных и смысл основных сущностей.
- `experiments/README.md` — как в проекте сейчас устроены эксперименты, тесты и регрессионные сценарии.
- `contracts_status.md` — текущее покрытие контрактов, черновые выводы и список спорных решений, которые нужно подтвердить перед заполнением `contracts/`.
- `TODO.md` — рабочий backlog проекта.

## Статус старых документов

- `RLPlatform_description.docx` оставлен в репозитории как архивный материал и источник части формулировок, но он не считается актуальной спецификацией.
- `database/Исходный код схемы базы данных dbml.docx` также носит справочный характер. Актуальная модель данных определяется текущими SQLAlchemy-моделями и миграциями Alembic.

## Краткая карта проекта

Сейчас платформа состоит из следующих рабочих частей:

- `apps/web` — React/Vite-клиент.
- `apps/api` — FastAPI-приложение с WebSocket-маршрутами и диспетчером экспериментов.
- `services/scenario_generator` — генерация, валидация и сериализация сценариев.
- `services/trail_camar`, `services/patrol_planning`, `services/reforestation_planting`, `services/simulator_3d` — адаптеры исполнения для разных режимов среды.
- `packages/db` — модель данных, сессии БД и миграции.
- `contracts/` — папка для формальных контрактов. На текущий момент покрыта частично; детали описаны в `contracts_status.md`.
- `data/` — сохранённые сценарии, реплеи и другие артефакты времени исполнения.

## Поддерживаемые режимы на текущий момент

- `continuous/trail`
- `discrete/patrol`
- `discrete/reforestation`
- `threed/patrol`
- `threed/trail`

Закомментированные маршруты и заготовки считаются незавершенными и не описываются как поддерживаемые публичные интерфейсы, пока не будут включены в код и зафиксированы в контрактах.
