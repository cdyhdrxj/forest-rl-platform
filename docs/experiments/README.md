# Эксперименты и проверки

## Что считается экспериментом в текущем проекте

Сейчас основной путь выполнения эксперимента проходит через `ExperimentDispatcher` в backend-сервисе:

1. генерируется или загружается сценарий;
2. создается `run`;
3. сервис исполнения запускается с параметрами обучения;
4. `RunObserver` пишет replay, метрики, эпизоды и события;
5. результаты остаются воспроизводимыми через связку `run + scenario_version + artifacts`.

Это основной поток, вокруг которого строится современная часть платформы.

## Каталог `experiments/`

На текущий момент каталог содержит легковесные экспериментальные и вспомогательные материалы:

- `benchmark_suites/` — скрипты для запуска базовых сравнений;
- `configs/` — место под конфигурации запусков;
- `notebooks/` — место под исследовательские ноутбуки;
- `quick_mode/` — легкие сценарии и smoke-проверки.

Часть подпапок пока является заготовкой, но сама структура уже отражает желаемое разделение между:

- воспроизводимыми benchmark-запусками;
- ad hoc исследованиями;
- быстрыми регрессионными проверками.

## Тесты как часть экспериментов

На практике важная часть проверки платформы сейчас живет в `tests/`.

### Что уже покрыто

- генерация сценариев;
- валидация generation request;
- сохранение и повторная загрузка сценария;
- сопоставление ROS-событий с типами событий платформы;
- интеграционный 3D flow через dispatcher;
- повторная загрузка `scenario_version` в новый `run`.

### Что это означает

Тесты уже проверяют ключевые свойства воспроизводимости, но еще не заменяют полноценный набор benchmark-протоколов.

## Быстрый и тяжелый режимы

В репозитории сосуществуют два типа окружений:

- лёгкие Python-сервисы исполнения, пригодные для быстрого тестирования и CI;
- задел под ROS 2 / 3D интеграцию для более реалистичных сценариев.

С точки зрения документации и контрактов важно, чтобы оба режима со временем использовали совместимые форматы сценариев, replay и метрик.

## Что пока не оформлено до конца

- единый каталог benchmark-конфигов;
- формальный протокол сравнения алгоритмов;
- набор golden scenarios;
- документы по MARL в `docs/experiments/marl/`.

Эти области пока считаются направлением развития, а не завершенной пользовательской документацией.
