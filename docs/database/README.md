# База данных и модель хранения

## Источник правды

Актуальная модель данных определяется не Word-документом, а кодом:

- `packages/db/models/*`
- `packages/db/migrations/*`

Файл `database/Исходный код схемы базы данных dbml.docx` сохранен как исторический материал, но уже расходится с текущей реализацией.
Например, в проекте уже есть режим `reforestation` и расширенный набор event types под ROS v2, чего в старом DBML-описании нет полностью.

## Основные группы сущностей

### Пользователи и проекты

- `users` — пользователи системы;
- `projects` — логические проекты;
- `algorithms` — каталог алгоритмов по mode/family.

### Сценарии

- `scenarios` — логическая сущность сценария;
- `scenario_versions` — версии сценария с seed, конфигами и ссылками на артефакты;
- `scenario_layers` — отдельные сохраненные слои среды.

### Запуски

- `runs` — конкретные исполнения сценария выбранным алгоритмом;
- `run_tags` — метки запуска;
- `service_logs` — инфраструктурные логи и логи времени исполнения backend-компонентов.

### Артефакты и модели

- `models` — модельные чекпойнты и связанные метрики;
- `artifacts` — зарегистрированные файлы запуска;
- `replays` — replay-файлы, связанные с run или episode.

### Аналитика выполнения

- `episodes` — агрегаты по эпизодам;
- `episode_events` — события внутри эпизодов;
- `metric_series` — описание серий метрик;
- `metric_points` — точки временных рядов.

## Как сейчас идет запись данных

### На этапе генерации сценария

Диспетчер:

- создает или переиспользует `project`, `scenario`, `algorithm`;
- записывает `scenario_version`;
- сохраняет `scenario_layers`;
- привязывает артефакты сценария к `run`.

### На этапе исполнения run

`RunObserver`:

- создает запись `replay`;
- пишет `Artifact` для replay-файла;
- сохраняет `MetricSeries` и `MetricPoint`;
- фиксирует завершенные `Episode`;
- пишет `EpisodeEvent`;
- обновляет `Run.status`, `started_at`, `finished_at`.

## Ключевые enum-значения

### `ProjectMode`

На текущий момент в коде присутствуют:

- `trail`
- `robot`
- `patrol`
- `fast_grid`
- `reforestation`

### `RunStatus`

- `created`
- `queued`
- `running`
- `finished`
- `failed`
- `cancelled`

### `ArtifactType`

- `world_file`
- `scenario_preview`
- `config`
- `log`
- `replay`
- `model_checkpoint`
- `metric_export`
- `report`
- `plot`
- `map_layer`

### `EventType`

В модели данных поддерживаются как новые типы событий ROS v2, так и исторические значения для обратной совместимости.
Это важно учитывать при формализации контрактов событий и миграций данных.

## Практический вывод

Если нужно понять, какие сущности реально существуют и как они связаны, смотреть нужно в SQLAlchemy-модели и миграции.
Word-файл полезен как историческая справка, но не как актуальная спецификация.
